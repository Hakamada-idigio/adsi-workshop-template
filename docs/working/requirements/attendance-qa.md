## 3. データ・ビジネスルール

[Question] 社員情報（社員ID）は自動採番ですか？それとも既存の社員番号を使いますか？
[Answer]
既存の社員番号を使います

[Question] 打刻の修正は誰ができますか？（例: 本人が申請→管理者承認 / 管理者が直接修正 / 修正不可）
[Answer]
承認前は本人がいつでも修正可能。承認後は本人が申請→管理者承認が必要。

[Question] 勤務時間の丸め処理は必要ですか？（例: 15分単位で切り捨て / 1分単位でそのまま / 会社の規定に合わせる）
[Answer]
1分単位でそのまま

## 4. 認証・権限

[Question] ログイン方式は？（例: ID＋パスワード / SSO連携 / OAuth）
[Answer]
ID＋パスワード

[Question] 権限のロール分けはどうしますか？（例: 一般社員・管理者の2種類 / 部門長も別ロール / フラット）
[Answer]
一般社員・管理者の2種類

## 5. 技術・制約

[Question] 今回のスコープとして、MVP（最小限の動くもの）としてどこまで作りますか？全機能を一度に作るか、段階的に作るか教えてください。
